# Analysis code — *Comparative Evaluation of Wearable Oxygen-Uptake Estimators Using Persistence, Transition, and Equivalence Analyses*

This repository contains all analysis code for the paper: preprocessing, model training,
hyperparameter tuning (including the reproducible Optuna scripts that regenerate the tuned
Random Forest and XGBoost under the 16-fold leave-one-participant-out (LOPO) RMSE objective),
the statistical analysis, and figure/table generation. Only the current, canonical scripts that
reproduce the reported results are included; superseded and exploratory scripts are not.

## Data availability

The raw dataset (`data/TB-File-01.csv`) is the publicly available dataset of **Beltrame, Amelard,
Wong & Hughson (2017),** *Prediction of oxygen uptake dynamics by machine learning analysis of
wearable sensors during activities of daily living*, **Scientific Reports 7:45738**,
https://doi.org/10.1038/srep45738 (CC BY 4.0). The original data collection was approved by the
Office of Research Ethics at the University of Waterloo (ID ORE20931). 16 participants, 1 Hz, the
matched analysis sample is 59,731 time points.

## Layout

```
data/                              TB-File-01.csv (public Beltrame 2017 dataset)
experiments/tuning/                linear training; temporal preprocessing/tuning/eval
experiments/pmea_r1_matched_tuning/ RF & XGBoost re-tuning (LOPO-RMSE) + linear tuning-robustness check
experiments/pmea_p0/               matched-master build; residual two-stage NULL result
experiments/pmea_p1/               statistics, equivalence/power, transition, agreement, MI, horizon, figures
experiments/analysis/              filter-cutoff sensitivity sweep
results/                           shipped intermediate artifacts (predictions, master, result tables)
figures/                           figures generated by the figure scripts (created on run)
```

## Reproduce

Run from the repository root. Steps 1–6 are GPU-heavy; the resulting predictions are **already shipped
under `results/`**, so the statistics, tables, and figures (steps 7–12) reproduce directly without re-training.

1. `python experiments/tuning/linear_models_perfold.py`  *(linear predictions)*
2. `python experiments/tuning/filter_seqlen_sweep.py`  *(temporal configs — GPU)*
3. `python experiments/tuning/optuna_temporal_tuning.py`  *(temporal tuning — GPU)*
4. `python experiments/tuning/final_evaluation.py`  *(temporal predictions — GPU)*
5. `python experiments/pmea_r1_matched_tuning/optuna_rf.py` and `optuna_xgb.py`  *(tree re-tuning)*
6. `python experiments/pmea_r1_matched_tuning/refresh_retuned.py`  *(re-tuned tree predictions)*
7. `python experiments/pmea_p0/p0_master_analysis.py`  *(builds the matched master from the predictions)*
8. `python experiments/pmea_p1/make_leaderboard_table.py`  *(model-comparison leaderboard — Table 2)*
9. `python experiments/pmea_p1/{p1_significance,p1_equivalence_power,causal_transition_window,bland_altman,horizon_uncertainty,mi_knn,detector_ci}.py`
10. `python experiments/pmea_p0/residual_final_eval.py` then `aggregate_residual.py`  *(residual NULL)*
11. `python experiments/analysis/filter_optimization.py`  *(filter sensitivity)*
12. `python experiments/pmea_p1/make_nature_figures.py` and `make_data_figures.py`  *(figures → `figures/`)*

## Notes

- **Tree provenance:** `p0_master_analysis.py` reads the re-tuned RF/XGB predictions
  (`results/pmea_r1_matched_tuning/{RF,XGB}/y_yhat.csv`), so re-running step 7 reproduces the canonical
  master with the LOPO-RMSE-tuned trees (verified: per-participant mean RMSE TCN 2.432, XGBoost 2.477,
  RF 2.507, Ridge 2.739).
- **Figures** are written to `./figures` (PNG + PDF; SVG archive under `results/pmea_p1/figures_nature/`).
- `results/filter_seqlen_sweep/best_configs.json` was reconstructed from the per-model
  `best_params.json` (the full Phase-1 sweep CSVs were not retained on disk); it holds the selected
  filter cutoff / sequence length per temporal model that fed Phase-2 tuning — GRU 0.1 Hz/120,
  LSTM 0.2 Hz/240, TCN 0.2 Hz/240, TFT 0.03 Hz/240, PatchTST 0.05 Hz/60. With it present, step 2 can be
  skipped when reproducing from the shipped predictions.
- Mutual information is computed in-script (10-bin histogram MI on the 0.01 Hz pooled low-pass features);
  conditional/unique MI is not reported.
- The reported **family-mean** Wilcoxon p-values (Temporal-vs-Linear/Tree, Tree-vs-Linear) are produced by
  `p1_significance.py`; `p04_family_wilcoxon.csv` from `p0_master_analysis.py` is the best-in-family upper-bound variant.
- `make_data_figures.py` regenerates the joint-distribution and raw-vs-filtered figures; the PRTS-protocol,
  input-feature, and model-validation supplementary figures are supplied as static originals (their helper
  functions are present but not invoked by default).
- The training/sweep scripts (`linear_models_perfold.py`, `filter_seqlen_sweep.py`, `optuna_temporal_tuning.py`,
  `final_evaluation.py`, `filter_optimization.py`) accept `--input data/TB-File-01.csv` and `--results results/<dir>`;
  pass these explicitly so outputs land under the canonical `results/` tree when run from the repository root.
